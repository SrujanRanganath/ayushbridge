- Request payload: query param `q` (min length 2)
- Response payload: array of NAMASTECode entries: [{ code, name, category, description, score? }]

- MongoDB collections: `NAMASTECode` (text index on `name`)
- Functional: Yes
- Production readiness: MEDIUM — text search exists; consider query sanitization, pagination, and rate limits. Missing explicit CORS/ACL per user.

4) POST /api/v1/map
- Request payload: { namasteCode, conditionName }
- Response payload (on cache hit): { namasteCode, icd11Code, icd11Title, confidence, source }
- Auth: Required
- MongoDB collections: `ICD11Mapping` (cache), `ICD11Code` (search)
- Functional: Yes (uses text index on `ICD11Code.title`), writes cache entry with confidence fixed at 0.85
- Production readiness: YELLOW — basic functionality is present, but concerns:
  - Confidence is hardcoded to 0.85 when caching (should be computed or calibrated)
  - No provenance/audit metadata captured on mapping cache entries
  - No fallback to external ICD-11 APIs if local DB fails
  - No rate limiting / throttling at per-user level beyond global limiter

5) POST /api/v1/fhir/bundle
- Request payload: { patientId, namasteCode, icd11Code, icd11Title, conditionName }
- Response payload: FHIR Bundle JSON (resourceType: Bundle) — server constructs a bundle containing Patient, Condition, Provenance
- Auth: Required
- MongoDB collections: `FHIRBundle` (stores bundleJSON and metadata)
- Functional: Yes — constructs valid-looking FHIR resources but not validated against a FHIR library/schema
- Production readiness: YELLOW/RED — concerns:
  - No FHIR schema validation or versioning (assumes structure is acceptable)
  - No patient existence checks (creates bundle referencing arbitrary patientId)
  - No consent or data provenance beyond a simple Provenance resource with static content
  - Potential for storing PII without encryption or access controls

6) GET /api/v1/dashboard
- Request payload: optional `page` and `limit` query params
- Response payload: { mappings: { data, total, page, pages }, bundles: { data, total, page, pages } }
- Auth: Required
- MongoDB collections: `ICD11Mapping`, `FHIRBundle`
- Functional: Yes
- Production readiness: YELLOW — pagination present, but response shape differs from what frontend expects in current client code (see frontend audit). Also, no filters or role-based access control.
