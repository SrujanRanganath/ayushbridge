Data flow (high level)
1. User searches on frontend (SearchPage) → client `searchService` → GET /api/v1/search (server) → MongoDB `NAMASTECode` text search → results.
2. User selects a NAMASTE term → MappingPage reads selection (sessionStorage or Router state) and triggers mapping. Frontend calling `mapNAMASTECode` → POST /api/v1/map → backend checks ICD11Mapping (cache) → if absent, text-search `ICD11Code` → writes ICD11Mapping cache → returns mapping result.
3. User may generate a FHIR Bundle → frontend calls `generateFHIRBundle` → POST /api/v1/fhir/bundle → server constructs FHIR Bundle JSON, stores in `FHIRBundle` collection, and returns Bundle JSON.
4. Dashboard page calls GET /api/v1/dashboard → server returns paginated ICD11Mapping and FHIRBundle records.
