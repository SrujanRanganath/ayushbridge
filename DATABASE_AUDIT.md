Database Audit — MongoDB models and indexes

Collections and models
- NAMASTECode (collection: namastecodes)
  - Fields: code (unique), name, category, description, createdAt
  - Indexes: text index on `name`
  - Notes: used by /search. No schema-level validation beyond Mongoose types.

- ICD11Code (collection: icd11codes)
  - Fields: code (unique), title
  - Indexes: text index on `title` (for full-text search)
  - Notes: used by /map route for text matching against conditionName.

- ICD11Mapping (collection: icd11mappings)
  - Fields: namasteCode (unique), icd11Code, icd11Title, confidence, cachedAt
  - Indexes: `namasteCode` unique constraint
  - Notes: serves as cache for mappings generated by /map

- FHIRBundle (collection: fhirbundles)
  - Fields: patientId, namasteCode, icd11Code, bundleJSON (mixed), generatedAt, auditLog
  - Indexes: none defined explicitly
  - Notes: stores returned bundle JSON; potential PII storage concerns

- AyurvedicEnrichment (collection: ayurvedicenrichments)
  - Fields: disease (unique), metadata fields; text index on `disease`

- User (collection: users)
  - Fields: name, email (unique), password (hashed), role

Indexes
- Text indexes exist for search usability on NAMASTECode.name and ICD11Code.title
- Unique indexes on code fields for NAMASTECode and ICD11Code
- No compound indexes for queries like recent mappings by date — could affect performance

Relations and validation
- Relationships are denormalized and stored as string references (e.g., FHIRBundle references patientId but no foreign key or ObjectId relationship)
- No JSON schema validation for stored FHIR bundles; bundle stored as mixed type

Missing/Weak areas
- Lack of indexes on generatedAt/cachedAt for sorting recent entries (server sorts but index absent)
- No TTL or archival policy for bundles or mapping caches
- No field-level validation for sensitive data stored in bundleJSON

Recommendations
- Add indexes on `FHIRBundle.generatedAt` and `ICD11Mapping.cachedAt` for efficient sorting
- Consider schema validation for `bundleJSON` using a FHIR schema or JSON schema
- Add storage/retention policies and mechanisms for PII
